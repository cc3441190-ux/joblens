**读者**：产品、研发、内测组织方

## PM 已确认：MVP 范围

| 维度 | 决策 |
|------|------|

| **明确不做** | **付费 / 商业化**（本阶段不排）；**PDF 批注**（仓库内 demo 可保留，不纳入 MVP 交付与宣传）。 |

## PM 已确认：迭代优先级（技术执行顺序）

由高到低：

1. **Side Panel 信息架构重构** — 首屏突出 **决策结论**（投/不投、匹配度、一条主因），其余模块 **Tab 折叠**，控制信息密度。  
2. **空状态与错误文案** — 抓取失败、无文本 JD、网络/模型错误、与列表不一致时的引导与话术。  
3. **北极星指标埋点** — 定义并落地可观测指标（与产品共同定口径后再开发）。  
4. **合规话术** — 免责声明、AI 边界、与平台关系（见下「对外话术」）。  
5. **多平台 / 商业化** — 本阶段 **不排**；仅在 MVP 稳定后再立项。

## PM 已确认：用户试用与发版门槛

- **内测规模**：**5 人**结构化内测（任务脚本 + 记录表）。  
- **验证重点**：**JD 与当前卡片一致性**、侧栏 **信息密度** 是否可理解、**Action**（如进入 Adapt、复制建议等）**转化率**。  
- **发版策略**：**内测通过后再推正式版本**（未通过则迭代 IA/文案/稳定性，再开一轮）。

## 1. 产品一句话（执行稿）

**JobLens**：面向 Boss 直聘 Web 的 Chrome 扩展，在 **不侵入 Boss 原生 DOM** 的前提下，用 **Side Panel** 完成「JD 透视 → 对照简历画像 → 基础 Adapt」闭环；API Key 仅存 Background，正文侧做手机号/邮箱等 **正则脱敏**。

---

## 2. 目标用户与核心场景

| 维度 | 说明 |
|------|------|
| 用户 | 校招生 / 初级职场人 |
| 主场景 | Boss 列表点选职位 → 侧栏 JD 透视 → 按需进入简历改写（Adapt） |
| 体验原则 | 零侵入；AI 提供 **Diff / 建议**，不越权代投 |

---

## 3. 当前实现与 MVP 差距（技术自检）

已实现：Onboarding、Boss 卡片点击 → JD 抓取与透视、简历画像、Adapt 链路、设计 token；已修 **SPA 切换导致 JD 与卡片错位** 及 **透视缓存 JD 指纹** 校验。

待对齐 MVP：侧栏 **IA 重构（首屏决策 + Tab）**、**空/错文案体系**、**指标埋点**、**法务话术落地**。

---

## 4. 技术边界（供排期参考）

- **架构**：Content Script + Side Panel + Background + `chrome.storage`；LLM 仅 Background 请求。  
- **JD**：同源 DOM + 详情 fetch 合并；依赖 Boss DOM，改版需跟进。  
- **风险残留**：快速连点多职位时，AI 在途请求的 **取消/排队** 仍可优化，与内测反馈一并排期。

---

## 5. 结论区（同步勾选）

- [x] **MVP 范围**：仅 Boss Web；JD 透视 + 简历画像 + 基础 Adapt；不做付费、不做 PDF 批注。  
- [x] **优先级**：IA 重构 > 空错文案 > 北极星埋点 > 合规话术 > 多平台/商业化（后者不排）。  
- [x] **发版**：5 人结构化内测（JD 一致性、信息密度、Action 转化）通过后上正式版。  
- [ ] **法务**：免责声明 +「与 Boss 官方无关联」定稿后替换产品内占位文案。
